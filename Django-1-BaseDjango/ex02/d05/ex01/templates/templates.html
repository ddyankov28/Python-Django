Understanding Django’s Default Template Engine

Django’s default template engine allows for a clean separation of the presentation layer (HTML) from backend logic. It supports various template tags, filters, and control structures for dynamically generating content.

Template Blocks

Django employs a block system for template inheritance. Blocks act as placeholder sections in a parent template that can be overridden by child templates. This promotes code reuse and adheres to the DRY (Don't Repeat Yourself) principle.

For example, a base template might define sections for the title, header, and content. Child templates can extend this base template and customize the content within these defined blocks.

Example Structure: The base template can include:

  • Title Block: Placeholder for page title.
  • Header Block: Placeholder for header content.
  • Content Block: Placeholder for the main content.

Loops

Django supports iterating through a list of items using a loop. This is useful for displaying collections such as blog posts or products. Within the loop, you can access each item to generate repeated elements dynamically.

Example Structure: To loop through a list, you might use a structure like:


    <ul>
        for each item in item_list:
            <li>{{ item }}</li>
    </ul>

If Control Structures

Django provides conditional logic capabilities using if statements in templates. This allows you to control what is displayed based on conditions, such as whether a user is logged in.

Example Structure: To display a message based on user authentication, you might structure it like this:


    if user is authenticated:
        <p>Welcome, {{ user.username }}!</p>
    else:
        <p>Please log in.</p>

Displaying Context Variables

Context variables are passed from views to templates, allowing dynamic content rendering. You can reference these variables to display their values directly in the HTML.

Example Structure: To display a greeting message and user information, you might use a structure like:


    <h1>{{ greeting }}</h1>
    <p>Current user: {{ user.username }}</p>
