How Django Serves Static Pages

1. User Request

The process begins when a user navigates to a specific URL in their web browser, sending an HTTP request to the server hosting the Django application.

2. URL Routing

Django's URL dispatcher routes the incoming request to the appropriate view based on the URL pattern defined in urls.py.

3. View Function

Once a matching URL pattern is found, Django calls the associated view function, which processes the request.

from django.shortcuts import render

def my_view(request):
    return render(request, 'my_template.html')

4. Template Rendering

The view function uses Django's template system to render an HTML page. The render function takes the HTTP request, template name, and context data.

5. Template Files

The specified template file (my_template.html) contains the HTML code. This file can include Django template language for dynamic content insertion.

<!-- In my_template.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>Welcome to my static page!</h1>
    <p>This is a simple static web page.</p>
</body>
</html>

6. Rendering Process

Django processes the template, replacing any template tags and variables with their corresponding values from the context, resulting in a fully rendered HTML string.

7. HTTP Response

The view returns an HTTP response object containing the rendered HTML, which is sent back to the user's web browser.

from django.http import HttpResponse

def my_view(request):
    return HttpResponse('<h1>This is a static page</h1>')

8. Displaying in the Browser

The user's browser receives the HTTP response and renders the HTML content, displaying the static web page to the user.
